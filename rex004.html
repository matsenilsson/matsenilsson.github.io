You have already used several R functions. Today we’ll take a closer look at how they work. After that we’ll continue working on the script: it’s finally time to do some statistical analyses of the result of our imaginary experiment.

 

Functions

Functions are small programs that do things. Most functions require that you specify what you want to be done by providing arguments, but some functions can be run without arguments:

date()
## [1] "Thu Nov 16 17:06:03 2017"

A function’s arguments is listed in its help page. Many functions have arguments with default values. For example, the mean() function has as default not to excluded missing data: na.rm = FALSE. This is a bit annoying, because R will give you an NA where you expected a number. But it is in fact helpful since it reminds you that your data contain missing values.

my_data <- rnorm(1000)
my_data[my_data > 2 ] <- NA  # Turn values > 2 into missing data (NA)
mean(my_data)  # The mean is NA if NAs are included
## [1] NA
mean(my_data, na.rm = TRUE)  # This gives the mean after removing NAs
## [1] -0.03293396

If you don’t specify what argument your are providing, R will assume that you’re entering them in the same order as specified by the function. For example, our favorite rnorm() has this order of arguments: rnorm(n, mean = 0, sd = 1).

So if you call rnorm(3, 100, 15), R will assume that you mean n = 3, mean = 100 and sd = 15 and give you three random numbers from N(100, 15).

set.seed(1)
rnorm(3, 100, 15)
## [1]  90.60319 102.75465  87.46557

Note that you can input them in another order if you specify each argument:

set.seed(1)
rnorm(sd = 15, n = 3, mean = 100)
## [1]  90.60319 102.75465  87.46557

When writing a script, it’s generally a good idea to specify arguments also when they are provided in the right order. This reminds you and others reading your code what the arguments stand for.

 

Writing your own functions

You can easily create your own functions. Here is an example of a function that takes a numeric vector as input and returns a z-transformed version of the data.

z_transform <- function(x) {
  zvalue <- (x - mean(x)) / sd(x)
  zvalue  # This returns the output 
}

When you run this code, it is stored as an object in R’s memory (you should see it in R-Studio’s Global Environment panel) and will be available until you close R or clear its workspace with rm(list = ls()) .

my_data <- c(1, 4, 9, 9, 14, 17)
my_z <- z_transform(my_data) 
my_z 
## [1] -1.3408030 -0.8380019  0.0000000  0.0000000  0.8380019  1.3408030

Note that the objects created inside a function (e.g., zvalue in z_transform()), are not available outside the function, so this will generate an error:

zvalue
## Error in eval(expr, envir, enclos): object 'zvalue' not found

Here is a user-made function that calculates sample statistics (three digits as default) and outputs these values in a named vector

calculate_stats <- function(x, digits = 3) {
  n <- length(x)
  m <- mean(x)
  md <- median(x)
  s <- sd(x)
  minimum <- min(x)
  maximum <- max(x)
  out <- round(c(n, m, s, md, minimum, maximum), digits)
  names(out) <- c('n', 'mean', 'sd', 'median', 'min', 'max')
  out
}

Here is another user-made function:

create_rsample <- function(n = 100, m = 100, s = 15) {
  # Creates a random sample of n observations with a specified mean (m) and 
  # standard deviation (s). The data is drawn from the normal distribution and
  # then linearly transformed to the specified mean and standard deviation.
  x <- rnorm(n)
  x <- (x - mean(x)) / sd(x)  # Sets mean(x) = 0 and sd(x) = 1
  x <- (x * s) + m  # Sets mean(x) = m and sd(x) = s
  x  # Return x
}

And here are both functions in action:

sample1 <- create_rsample()
calculate_stats(sample1)
##       n    mean      sd  median     min     max 
## 100.000 100.000  15.000  99.461  61.264 138.291
sample2 <- create_rsample(10, 50, 10)
calculate_stats(sample2)
##      n   mean     sd median    min    max 
## 10.000 50.000 10.000 50.221 37.242 63.329

If these were your first functions: Congratulations, you just wrote your first computer programs!

Note that it’s a very good idea to comment your code carefully, including your functions (as I did above for create_rsample()). The time spend on commenting your code is usually much less then the time you will need to understand your uncommented code when revisited on another day.

 

Exercises

This code creates an integer vector randomly drawn with replacement from the set of integers between 1 and 10.

set.seed(123)
randint <- sample(seq(1, 10), 10, replace = TRUE)
randint
##  [1]  3  8  5  9 10  1  6  9  6  5

Create a function that takes a vector created in this way as input and returns a vector with three statistics: (i) the number of unique integers, (ii) their mean value (excluding the duplicates), and (iii) the mean of the non-included integers between 1 and 10.
If the randint vector defined above would be given as input to this function, something like this would be returned:

##         #included     mean-included mean-not-included 
##          7.000000          6.000000          4.333333
